# link-keeper 記事登録スキル

残しておきたい記事URLを **link-keeper**（`~/Projects/link-keeper`）の Markdown 蓄積簿へ、一貫した分類で登録する。複数URLをまとめて渡された場合も順に処理する。

## 使用タイミング

- 記事URL（1件〜複数）を渡して「登録して」「link-keeper に追加して」と言われた時
- 「この記事も追加」「以下も登録」とURLが列挙された時

SKIP: link-keeper 以外への登録、登録済みURLの再登録の時。

## データ構造

データの正本は `docs/リソース蓄積簿.md`。Markdown テーブルに以下の列を持つ:

| 列名 | 必須 | 説明 |
|---|---|---|
| タイトル | 必須 | 記事の正式タイトル |
| URL | 必須 | 記事の URL |
| 説明 | 必須 | 1〜2 行の概要（捏造しない） |
| メモ | 任意 | 一言メモ（空でもよい） |
| カテゴリ | 必須 | 知識 / 情報 |
| 種類 | 必須 | 記事 / リポジトリ / 書籍 / 動画 / ツール / テンプレート |
| 話題 | 必須 | 語彙定義セクションの話題テーブルから選択 |

## 基本ワークフロー

### Phase 1: 重複確認

蓄積簿に同一 URL が既に登録されていないか確認する。

```bash
grep -F '<URL>' ~/Projects/link-keeper/docs/リソース蓄積簿.md
```

ヒットしたら「登録済みです」と報告して終了。

### Phase 2: タイトル取得

各 URL の `og:title`（無ければ `<title>`）を取得する。WebFetch で取得を試みる。取得できなければタイトルを手動で推定し、その旨を報告する。

### Phase 3: 分類と説明文の決定

「分類ルール」節に従い、カテゴリ・種類・話題を決める。説明文（1〜2 行）は簡潔に作成する（タイトルだけで断定できない詳細は書かない）。

### Phase 4: 蓄積簿に追記

`docs/リソース蓄積簿.md` の `## リソース一覧` テーブルの末尾に行を追加する。

Edit ツールで、テーブルの最終行の直後に新しい行を追加する:

```
| <タイトル> | <URL> | <説明> | <メモ> | <カテゴリ> | <種類> | <話題> |
```

注意: セル内のパイプ文字 `|` は避ける（Markdown テーブルが壊れるため）。

### Phase 5: ビルド実行

蓄積簿の変更を index.html に反映する。

```bash
cd ~/Projects/link-keeper && npm run build
```

「build 完了: N 件のリソースを埋め込みました」と出力されることを確認。

### Phase 6: 検証と報告

ビルド後の件数が追加前 +1 であることを確認し、ユーザーへ表形式で報告する。

```bash
grep -c '"title"' ~/Projects/link-keeper/index.html
```

報告形式:

| 項目 | 値 |
|---|---|
| タイトル | <タイトル> |
| URL | <URL> |
| カテゴリ | <カテゴリ> |
| 種類 | <種類> |
| 話題 | <話題> |
| 登録後の総件数 | N 件 |

## 完了条件

| Phase | 完了条件 |
|---|---|
| Phase 1 | URL の重複がないことを確認 |
| Phase 2 | タイトルが取得できている |
| Phase 3 | カテゴリ・種類・話題・説明文が決定している |
| Phase 4 | 蓄積簿に行が追加されている |
| Phase 5 | ビルドが成功し件数が増えている |
| Phase 6 | ユーザーに報告済み |
| **Goal** | 記事が蓄積簿に登録され、index.html に反映されている |

## 分類ルール

### カテゴリ（「1年後にも使えるか」で二分）

- **情報**: 進化の速い AI/特定ツールの知見・速報・事例（1年後には価値が下がるもの）
- **知識**: ツール非依存の普遍的な考え方・枠組み（durable な技術）
- 迷う場合は判断と理由を添えて報告する

### 種類（ResourceType）

Web 記事は基本「記事」。GitHub リポジトリは「リポジトリ」。Web ツールは「ツール」。

### 話題（Topic・必須）

蓄積簿の語彙定義セクションから全件取得し、記事の主題に最も近い 1 つを選ぶ。迷ったら記事タイトルの主語で判断する。

## 重要な注意事項

- **説明文は捏造しない**: タイトルで保証できない具体は書かない。本文を読めた時のみ踏み込む
- **同一URLの重複登録を避ける**: Phase 1 で必ず確認する
- **パイプ文字を避ける**: テーブルセル内に `|` があると Markdown が壊れる。別の表現に置き換える
